Question 24

Discuss how memory management is dealt with in Linux operating system. (10 Marks)

Answer:

28.0 memory management

· Linux's physical memory-management system deals with allocating and freeing pages, groups of pages, and small blocks of memory.

· It has additional mechanisms for handling virtual memory, memory mapped into the address space of running processes.

Splitting of Memory in a Buddy Heap



28.1 Managing Physical Memory

· The page allocator allocates and frees all physical pages; it can allocate ranges of physically-contiguous pages on request.

The allocator uses a buddy-heap algorithm to keep track of available physical pages.

i. Each allocatable memory region is paired with an adjacent partner.

ii. Whenever two allocated partner regions are both freed up they are combined to form a larger region.

iii. If a small memory request cannot be satisfied by allocating an existing small free region, then a larger free region will be subdivided into two partners to satisfy the request.

· Memory allocations in the Linux kernel occur either statically (drivers reserve a contiguous area of memory during system boot time) or dynamically (via the page allocator).

28.2 Virtual Memory

· The VM system maintains the address space visible to each process: It creates pages of virtual memory on demand, and manages the loading of those pages from disk or their swapping back out to disk as required.

The VM manager maintains two separate views of a process's address space:

i. A logical view describing instructions concerning the layout of the address space. The address space consists of a set of nonoverlapping regions, each representing a continuous, page-aligned subset of the address space.

ii. A physical view of each address space which is stored in the hardware page tables for the process.

Virtual memory regions are characterized by:

i. The backing store, which describes from where the pages for a region come; regions are usually backed by a file or by nothing ( demand-zero memory)

ii. The region's reaction to writes (page sharing or copy-on write).

The kernel creates a new virtual address space

i. When a process runs a new program with the

exec

system call

ii. Upon creation of a new process by the

fork

system call

· On executing a new program, the process is given a new, completely empty virtual- address space; the program loading routines populate the address space with virtual memory regions.

Creating a new process with

fork

involves creating a complete copy of the existing process's virtual address space.

i. The kernel copies the parent process's VMA descriptors, then creates a new set of page tables for the child.

ii. The parent's page tables are copies directly into the child's, with the reference count of each page covered being incremented.

iii. After the fork, the parent and child share the same physical pages of memory in their address spaces.

· The VM paging system relocates pages of memory from physical memory out to disk when the memory is needed for something else.

The VM paging system can be divided into two sections:

i. The pageout-policy algorithm decides which pages to write out to disk, and when.

ii. The paging mechanism actually carries out the transfer, and pages data back into physical memory as needed.

· The Linux kernel reserves a constant, architecture dependent region of the virtual address space of every process for its own internal use.

This kernel virtual-memory area contains two regions:

i. A static area that contains page table references to every available physical page of memory in the system, so that there is a simple translation from physical to virtual addresses when running kernel code.

ii. The reminder of the reserved section is not reserved for any specific purpose; its page-table entries can be modified to point to any other areas of memory.
